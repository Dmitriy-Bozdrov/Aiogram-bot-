Вот подробное техническое задание (ТЗ) для программиста:

⸻

Техническое задание: Telegram-бот для приёма USDT (TRC20 и ERC20)

1. Цель проекта

Создать Telegram-бота, через которого клиенты смогут удобно отправлять USDT на указанные адреса в сетях TRC20 и ERC20. Бот должен проверять поступление средств, уведомлять оператора/админа и клиента, фиксировать данные в лог.

⸻

2. Основной функционал
 1. Приветствие и выбор языка (UA/RU/EN).
 2. Выбор сети перевода:
 • TRC20 (сеть Tron)
 • ERC20 (сеть Ethereum)
 3. Выдача актуального адреса для приёма USDT:
 • Автоматически сгенерированного (из пула кошельков или API).
 • Либо фиксированный адрес, если один кошелек на сеть.
 4. Проверка поступления транзакции:
 • Использование стороннего API или собственного нода.
 • Автоматическое отслеживание входящих транзакций.
 • Подтверждение перевода (1 подтверждение TRC20, 6 для ERC20 по умолчанию).
 5. Уведомления:
 • Клиенту — об успешной оплате (с указанием суммы, хеша и времени).
 • Администратору — об успешной оплате и данных пользователя.
 6. Интеграция с Google Таблицей / CRM (опционально):
 • Запись данных о каждой транзакции.
 7. Реферальный механизм (опционально):
 • Присвоение реферального ID и отслеживание, кто привёл клиента.

⸻

3. Технические требования
 • Язык: Python (aiogram / pyrogram) или Node.js (telegraf.js).
 • Хостинг: VPS / сервер с 24/7 uptime.
 • Подключение API:
 • TRONSCAN API / TRONGRID для TRC20
 • ETHERSCAN или INFURA для ERC20
 • Поддержка мультисетевых кошельков (TRC20 и ERC20).
 • Безопасное хранение приватных ключей или использование стороннего кошелька с API (например, NowNodes, GetBlock, Tatum, Crypto APIs и т.д.)

⸻

4. Панель администратора (в Telegram или через Google Таблицу)
 • Просмотр логов по транзакциям.
 • Управление кошельками (добавить/удалить).
 • Настройка комиссий / курсов (если нужно показывать).

⸻

5. Требования по безопасности
 • Ограничение доступа к API-ключам.
 • Работа с вебхуками через HTTPS.
 • Защита от повторной отправки одних и тех же данных (idempotency).

Позволить клиенту оставить заявку на внесение наличных (USD или UAH) в кассу обменного пункта. Заявка отправляется менеджеру вручную, без автоматического подтверждения.

⸻

Пошаговый процесс:
 1. Клиент выбирает опцию:
“Внести наличные в кассу”

 • USD
 • UAH
 3. Ввод суммы вручную
 4. Выбор города и отделения (если есть несколько):
 • Например:
 • Днепр, просп. Гагарина
 • Львов, ул. Зеленая
 5. Выбор удобного времени визита:
(например, через быстрые кнопки: сегодня / завтра, утро / день / вечер)
 6. Ввод имени и номера телефона (или выбор из Telegram профиля)
 7. Заявка формируется и уходит менеджеру:
 • Внутреннее сообщение в Telegram (в админ-группу или в чат менеджера).
